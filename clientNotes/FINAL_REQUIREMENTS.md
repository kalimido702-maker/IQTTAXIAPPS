## 🐛 1. Bugs - مشاكل يجب إصلاحها فوراً

### BUG-01: حساب الأجرة يعرض قيم صفرية [HIGH]
- **الوصف**: شاشة تفاصيل الرحلة تعرض `KM 0.0` لسعر المسافة الأساسية والإضافية والضرائب `0%`، رغم أن الإجمالي يظهر مبلغ (3935 IQD / 3000 IQD)
- **التطبيق**: Passenger App
- **الشاشة**: `TripDetailsPage` (feature: `trip/`)
- **المرجع**: `-0000009-PHOTO-2026-02-25`, `-0000021-PHOTO-2026-02-25`
- **المطلوب**: مراجعة `fare_breakdown` Logic في الـ backend والـ frontend - التأكد من أن API يرجع القيم الصحيحة وأن الـ UI يعرضها

### BUG-02: خطأ API Key يحتوي على whitespace [CRITICAL]
- **الوصف**: ظهور شريط خطأ أحمر `"api_key cannot contain whitespace"` في شاشة المحفظة/التحويلات
- **التطبيق**: Both
- **الشاشة**: `WalletPage` (feature: `wallet/`)
- **المرجع**: `-0000020-PHOTO-2026-02-25`
- **المطلوب**: فحص كيفية تخزين وقراءة الـ API key - على الأرجح يوجد space أو newline في الـ key المخزن في `.env` أو الـ backend config

### BUG-03: تعارض في رسالة إلغاء الرحلة [HIGH]
- **الوصف**: الـ notification يقول "تم إلغاء الرحلة من قبل العميل" لكن الـ body يقول "ألغى السائق الرحلة" - تناقض واضح يُربك المستخدم
- **التطبيق**: Passenger App
- **الشاشة**: Push Notification + `NotificationPage` (feature: `notification/`)
- **المرجع**: `00000001-PHOTO-2026-03-01`
- **المطلوب**: مراجعة الـ notification payload من الـ backend - يجب أن يتم إرسال الرسالة الصحيحة حسب من قام بالإلغاء (سائق/راكب). مراجعة `AppStrings` للتأكد من وجود رسائل منفصلة لكل حالة

### BUG-04: خطأ إملائي "transfered" → "transferred" [LOW]
- **الوصف**: كلمة `transferred` مكتوبة بشكل خاطئ `transfered` في شاشة المحفظة
- **التطبيق**: Both
- **الشاشة**: `WalletPage` (feature: `wallet/`)
- **المرجع**: `-0000014-PHOTO-2026-02-25`
- **المطلوب**: تصحيح في `AppStrings` - البحث عن `transfered` واستبداله بـ `transferred`

### BUG-05: نصوص غير مترجمة في Admin Panel [MEDIUM]
- **الوصف**: ظهور keys خام مثل `driver_bulk_upload`، `free_days_description`، `(can_select_multiple)` بدلاً من النصوص المترجمة
- **التطبيق**: Admin Panel (Web)
- **المرجع**: `00000036-PHOTO-2026-03-08`
- **المطلوب**: إضافة الترجمات الناقصة في ملفات الـ localization للـ Admin Panel

### BUG-06: رسالة نجاح التسجيل بالإنجليزية فقط [MEDIUM]
- **الوصف**: عند التسجيل تظهر رسالة `"User registered successfully. Please verify OTP..."` بالإنجليزية فقط - المفروض تكون بالعربية (اللغة الأساسية)
- **التطبيق**: Both
- **الشاشة**: `RegisterPage` (feature: `auth/`)
- **المرجع**: `-0000023-PHOTO-2026-02-25`
- **المطلوب**: التأكد من أن الـ snackbar messages تستخدم `AppStrings` المترجمة وليس نصوص hardcoded

## ✨ 2. Features - ميزات جديدة مطلوبة

### FEAT-01: تحسين نظام المواقع المفضلة [HIGH]
- **الوصف**: شاشة "الموقع المفضل" تحتاج تحسينات كبيرة:
  - (أ) منع تكرار أسماء المواقع (ظهور "البيت" مرتين)
  - (ب) إمكانية تعديل المواقع غير المحددة (العمل يظهر "لم يتم تحديد الموقع بعد" بدون طريقة واضحة لتحديده)
  - (ج) تحسين UX لإضافة عنوان جديد (الفيديو يوضح أن العملية تعمل لكن تحتاج polish)
- **التطبيق**: Passenger App
- **الشاشة**: `FavouriteLocationPage` (feature: `favourite_location/`)
- **المرجع**: `-0000003`, `-0000015`, `-0000018`, `-0000004-VIDEO`
- **مهام فرعية**:
  1. إضافة validation لمنع تكرار الأسماء
  2. عند الضغط على موقع "لم يتم تحديده" ينقل المستخدم لشاشة اختيار الموقع
  3. إضافة معاينة صغيرة للخريطة في شاشة حفظ العنوان الجديد

### FEAT-02: تحسين مشاركة رمز الدعوة (Referral) [MEDIUM]
- **الوصف**: شاشة "دعوة صديق واربح" تحتاج:
  - (أ) عرض رمز الدعوة بشكل واضح (حالياً يوجد زر نسخ فقط بدون عرض الرمز)
  - (ب) مشاركة مباشرة عبر WhatsApp/SMS/etc بدلاً من النسخ فقط
  - (ج) شرح واضح لكيفية عمل نظام المكافآت
- **التطبيق**: Both
- **الشاشة**: `ReferralPage` (feature: `referral/`)
- **المرجع**: `-0000011`, `-0000019`
- **مهام فرعية**:
  1. عرض الكود بشكل مرئي في الشاشة
  2. استخدام `Share.share()` من package `share_plus` لمشاركة native
  3. إضافة قسم "كيف تعمل" يشرح خطوات الربح

### FEAT-03: زر تأكيد في شاشة إضافة رصيد [MEDIUM]
- **الوصف**: شاشة "إضافة رصيد" تعرض أزرار اختيار المبلغ (5K, 10K, 15K IQD) وحقل إدخال، لكن لا يوجد زر تأكيد واضح أو زر إلغاء
- **التطبيق**: Both
- **الشاشة**: `AddBalancePage` (feature: `wallet/`)
- **المرجع**: `-0000012`
- **المطلوب**: إضافة زر "تأكيد الإضافة" وزر "إلغاء" واضحين

### FEAT-04: تفاصيل المعاملات في المحفظة [LOW]
- **الوصف**: قائمة المعاملات الأخيرة في المحفظة تعرض معلومات أساسية فقط - يفضل إضافة إمكانية الضغط لعرض تفاصيل كل معاملة
- **التطبيق**: Both
- **الشاشة**: `WalletPage` (feature: `wallet/`)
- **المرجع**: `-0000014`

### FEAT-05: نظام تأكيد استلام المبلغ النقدي [HIGH]
- **الوصف**: في شاشة ملخص الرحلة يظهر "بانتظار تأكيد السائق لاستلام المبلغ" - وهذا يحتاج مراجعة لضمان عدم تعليق الرحلة إذا لم يؤكد السائق
- **التطبيق**: Both (Passenger shows waiting, Driver confirms)
- **الشاشة**: `TripSummaryPage` (feature: `trip/`)
- **المرجع**: `00000047-PHOTO-2026-03-11`
- **المطلوب**: مراجعة flow تأكيد الدفع + إضافة timeout تلقائي + إشعار reminder للسائق

## 🎨 3. UI/UX - تعديلات في التصميم وتجربة المستخدم

### UI-01: تقطيع النصوص في عناوين الرحلات [HIGH]
- **الوصف**: عناوين نقاط البداية والنهاية في تفاصيل الرحلة تتقطع لأنها طويلة جداً - تحتاج `overflow: ellipsis` أو `scrollable` text
- **التطبيق**: Passenger App
- **الشاشة**: `TripDetailsPage`
- **المرجع**: `-0000009`, `-0000021`
- **المطلوب**: استخدام `maxLines` + `TextOverflow.ellipsis` مع إمكانية الضغط لعرض العنوان الكامل

### UI-02: تحسين أيقونات أنواع السيارات في الخريطة [MEDIUM]
- **الوصف**: أيقونات أنواع السيارات (تاكسي، مندوب، VIP، محافظات) متشابهة جداً - كلها نفس صورة السيارة مع اختلاف طفيف
- **التطبيق**: Passenger App
- **الشاشة**: `HomePage` (feature: `home/`)
- **المرجع**: `-0000016`, `-0000022`
- **المطلوب**: تصميم أيقونات مختلفة بوضوح لكل نوع خدمة (تاكسي عادي، VIP بسيارة فاخرة، مندوب بصندوق توصيل، محافظات بأيقونة مسافة)

### UI-03: تنظيم أفضل لتفاصيل الأجرة [MEDIUM]
- **الوصف**: تفاصيل الأجرة في شاشة الرحلة تحتاج تنظيم أفضل:
  - فصل واضح بين المكونات (أجرة أساسية، مسافة إضافية، ضرائب)
  - عرض النسب المئوية بشكل واضح
  - إجمالي بارز
- **التطبيق**: Passenger App
- **الشاشة**: `TripDetailsPage`, `TripSummaryPage`
- **المرجع**: `-0000009`, `-0000021`, `00000047`

### UI-04: صفحة الإعدادات فقيرة المحتوى [LOW]
- **الوصف**: صفحة الإعدادات تحتوي فقط على: Dark mode toggle + التعليمات + سياسة الخصوصية + تسجيل خروج. تحتاج إضافات:
  - تغيير اللغة
  - إدارة الحساب
  - الإشعارات
  - نسخة التطبيق
- **التطبيق**: Both
- **الشاشة**: `SettingsPage` (feature: `settings/`)
- **المرجع**: `-0000010`, `-0000017`

### UI-05: تحسين شريط البحث في الصفحة الرئيسية [LOW]
- **الوصف**: شريط البحث "إلى أين أنت ذاهب؟" يحتاج تصميم أكثر بروزاً وتفاعلية
- **التطبيق**: Passenger App
